Settings Guide

How to configure each section in the Lucid settings panel.

1 Lucid REST API

Connects to the Lucid.co visual collaboration platform. The API key is a Bearer token for authenticating with https://api.lucid.co.

How to get an API key

  1. Go to lucid.app/developer → API Keys.
  2. Click Create API Key.
  3. Set a name and expiration date for the key.
  4. Set the grants (permissions) as shown in the table below.
  5. Click Generate API key and save it securely — you won't see it again.
  6. Paste the key into the Lucid REST API field in Settings.

Required Grants

Grant Set To Why
Documents Read and Write Required to create diagrams via the Standard Import API (POST /documents).
Users None Optional — not required for core functionality.
Folders None Optional — set to Read if you want to target a specific folder.
Accounts None Not needed.
Teams None Not needed.
Requires a licensed Lucid account with developer tools unlocked. Grants cannot be changed after creating a key — create a new key if you need different permissions.

📖 Full documentation: developer.lucid.co/docs/api-keys

2 Active AI Provider

This radio selector chooses which AI provider processes your uploaded images. Only one provider can be active at a time.

  1. Open Settings and expand the Active AI Provider section.
  2. Select the provider you want to use (Gemini, OpenAI, Claude, or xAI).
  3. Enter the API key for the selected provider in its own section below.
  4. Click Save Settings.
You must also enter the API key for your selected provider — choosing a provider alone is not enough. The navbar status indicator checks connectivity for the selected provider only.

3 AI Provider API Keys

You only need an API key for the one provider you've selected above. Enter it in the matching section in Settings.

Provider Where to get a key
Gemini (Google) aistudio.google.com/apikey
OpenAI platform.openai.com/api-keys
Claude (Anthropic) console.anthropic.com/settings/keys
xAI (Grok) console.x.ai
Only one AI provider is used at a time. You can enter keys for multiple providers and switch between them in the Active AI Provider section.

4 Timeouts Read-only

These values are set in config.json at build time and cannot be changed from the UI.

Setting Description Default
AI API Timeout Maximum seconds to wait for an AI provider response. 30
Lucid API Timeout Maximum seconds to wait for the Lucid REST API response. 15
To change these values, edit config.json and rebuild the Docker container.

5 Couchbase Lite Read-only

Couchbase Lite is the local embedded database that runs inside the Docker container. It stores uploaded image metadata and AI results locally.

No setup needed — Couchbase Lite is configured automatically when the container starts.
Stat Description
Status Active = the CBL library loaded successfully. Fallback = CBL is unavailable and the app is using JSON files for storage instead.
Database The name of the Couchbase Lite database in use.
Document Count Total number of documents stored locally (images, metadata, etc.).
Path The filesystem path inside the container where the database files are stored.

6 Status Indicators

The three dots in the navbar show real-time connectivity status. They refresh automatically every 30 seconds.

Indicator 🟢 Green 🔴 Red
CBL Couchbase Lite library loaded and active. CBL not available; app is using fallback storage.
AI API Selected AI provider is reachable and responding. Unreachable, or no API key set for the selected provider.
Lucid Lucid REST API authenticated and responding. Unreachable, or no API key set.
If a dot is red, open Settings and verify the corresponding API key is entered correctly.
